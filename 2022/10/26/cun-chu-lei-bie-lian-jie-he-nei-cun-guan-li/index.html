一、存储类别

概念

  • 对象：一块指定的内存
  • 标识符：名称，用于指定特定对象（一块内存）的内容，方便访问对象
  • 左值：指定对象的表达式 
    eg.
int entity = 3   //entity(标识符/左值）指定一块内存（对象）为3（内容）
int* pt = "abc"  //pt(标识符/左值）指定一块内存存储地址，*pt(表达式/左值）指定一块内存存"abc"

    2）作用域——描述程序中可访问标识符的区域

  • 块作用域
    double blocky(double cleo)
{
	double patrick = 0.0;
	int i;
	for (i = 0; i < 10; i++)
	{
		double q = cleo * i;	//q的作用域开始
		patrick *= q;
	}				//q的作用域结束
	return patrick;
}
  • 函数作用域
    仅用于goto语句的标签，即是一个标签首次出现在函数的内层块中，它的作用域也延伸至整个函数，防止两个块中使用同一标签而显得混乱。
  • 函数原型作用域
    用于函数原型中的形参名，范围是从形参定义处到原型声明结束。
  • 文件作用域
    从变量的定义处到文件末尾
    #include<stdio.h>	
int units = 0;　　//该变量具有文件作用域
int main()
{

}

    Tip： 翻译单元和文件
    编译器把源代码文件和所有的头文件看作一个包含信息的单独文件。这个单独文件被称为翻译单元。
    描述一个具有文件作用域的变量时，它的实际可见范围是整个翻译单元。如果程序由多个源代码文件组成，
    那么该程序也将由多个翻译单元组成。每个翻译单元均对应一个源代码文件和它所包含的文件。

3）链接

  • 种类：外部链接，内部链接和无链接
    int giants = 5;　 //外部链接的文件作用域，能跨翻译单元
static int dodgers = 3;　　　　//内部链接的文件作用域，不能跨翻译单元
　　　　　　　　　　　　　　　　//static是存储类别说明符  
int main()
{

}

    4)存储期

  • 静态存储期：在程序执行期间一直存在。文件作用域变量具有静态存储期。
  • 线程存储期：从声明到线程结束，每个线程都获得该变量的私有备份。
  • 自动存储期：块作用域的变量一般具有自动存储期。从块的开始，为这些变量分配内存，到块的结束，释放刚才分配的内存。

    Tip:
    1.变长数组的存储期是从声明处到块的末尾。
    2.块作用域变量也能具有静态存储期。

    void more(int number)
{
	int index;
	static int ct = 0;　/*只有在执行该函数时,能通过ct(标识符）访问其指定的对象，
	                      但也可以通过地址，直接访问对象*/
	return = 0;
}

    5)五种存储类别

    存储类别 存储期 作用域 链接 声明方式
    自动 自动 块 无 块内
    寄存器 自动 块 无 块内，使用关键字register
    静态外部链接 静态 文件 外部 所有函数外
    静态内部链接 静态 文件 内部 所有函数外，使用关键字static
    静态无链接 静态 块 无 块内，使用关键字static

    6)自动变量

    属于自动存储类别的变量具有自动存储期、块作用域且无链接。可以使用存储类别说明符auto声明。
    // hiding.c -- variables in blocks
#include <stdio.h>
int main()
{
    int x = 30;      // 原始 x
    
    printf("x in outer block: %d at %p\n", x, &x);
    {
        int x = 77;  // 新的x, 隐藏了原始的x
        printf("x in inner block: %d at %p\n", x, &x);
    }
    printf("x in outer block: %d at %p\n", x, &x);
    while (x++ < 33) // 原始的x
    {
        int x = 100; // 新的x，隐藏了原始的x 
        x++;
        printf("x in while loop: %d at %p\n", x, &x);
    }
    printf("x in outer block: %d at %p\n", x, &x);

    return 0;
}

    7)寄存器变量

    用关键字register声明。存储在CPU的寄存器中，或者说，存储在最快的可用内存中。与普通变量相比，访问和处理这些变量的速度更快，
    但无法获取变量地址。绝大多数方面，寄存器变量与自动变量一样。

    8)块作用域的静态变量

    静态变量指变量的内存原地不动。具有文件作用域的变量一定是静态变量。具有块作用域的变量，可以用static声明其为静态变量。

    9）外部链接的静态变量

    外部链接的静态变量具有文件作用域、外部链接和静态存储期。可以选择在函数中用extern再次声明。
    如果要在另一个源代码文件中使用该变量，则必须在该文件中使用extern再次声明。
  1. 初始化外部变量
  • 未初始化的外部变量自动初始化为0。
  • 外部变量只能用常量初始化

  1. 定义与声明的区别

    int tern = 1;//tern被定义
main()
{
    extern int tern;//使用在别处定义的tern
}

    这里，tern被声明了两次。第一次为定义式声明，为变量预留了存储空间。第二次为引用式声明，
    它指示编译器去别处查找其定义，并不为其分配内存。

    10）内部链接的静态变量

    该变量具有文件作用域、内部链接和静态存储期。在所有函数外，用static定义该变量。
    可以用extern在函数中重复声明任何具有文件作用域的变量，但并不会改变其链接属性。

    11）存储类别说明符

  2. auto:只用于块作用域的变量声明中，明确表达要使用与外部变量同名的局部变量

  3. register:只用于块作用域的变量声明中，请求最快速度访问该变量，同时，还保护了该变量的地址不被获取。

  4. static:说明创建的对象存储期为静态存储期，但不改变其作用域。
    因此，只有在块中，这种变量才能通过标识符访问。在其他块，可以通过地址访问。

  5. extern：表明声明的变量定义在别处。

    12）存储类别和函数

    函数的存储类别：外部函数（默认），静态函数，内联函数。

    二、随机函数和静态变量

    rand()函数

  6. rand()不需要参数，它会返回一个从0到最大随机数的任意整数，最大随机数的大小通常是固定的一个大整数。

  7. 如果你要产生099这100个整数中的一个随机整数，可以表达为：int num = rand() % 100; 这样，num的值就是一个099中的一个随机数了。

  8. 如果要产生1~100，则是这样：int num = rand() % 100 + 1;

  9. 总结来说，可以表示为：int num = rand() % n +a;
    其中的a是起始值，n-1+a是终止值，n是整数的范围。

  10. 一般性：rand() % (b-a+1)+ a ; 就表示 a~b 之间的一个随机整数。

    • 若要产生0-1之间的小数，则可以先取得0-10的整数，然后均除以10即可得到“随机到十分位”的10个随机小数。
    • 若要得到“随机到百分位”的随机小数，则需要先得到0~100的10个整数，然后均除以100，其它情况依 此类推。
